# E4-R0 与 E0 首 50 题对比分析

## 1. 比较口径

- E4-R0：`results/e4_r0_run1.json`，完整 50 题；
- E0：取 `e0_core_run1.json` 和 `e0_core_run2.json` 中与 E4-R0 相同的前 50 个题目；
- 模型、few-shot、题目选择和 `--limit 50` 口径保持一致；
- E4-R0 只增加 capability gate，不启用 strict verified-final；
- 由于 E4-R0 只有一次运行，本比较只能说明探索性趋势，不能估计 E4-R0 的运行方差。

## 2. 总体结果

| 指标 | E0 run1 | E0 run2 | E0 首 50 均值 | E4-R0 | E4-R0 相对均值 |
|---|---:|---:|---:|---:|---:|
| 正确数 | 19/50 | 20/50 | 19.5/50 | 19/50 | -0.5 题 |
| 准确率 | 38.00% | 40.00% | 39.00% | 38.00% | -1.00 pp |
| Total tokens/题 | 10,903.32 | 12,107.40 | 11,505.36 | 10,207.28 | -11.29% |
| LLM 调用/题 | 2.66 | 2.86 | 2.76 | 2.72 | -1.45% |
| 延迟/题（秒） | 35.10 | 40.19 | 37.64 | 36.69 | -2.54% |

E4-R0 的准确率处于 E0 两次运行的历史范围内，成本略低于 E0 两次运行的
平均值。由于只有一次 E4-R0 运行，不能把这点成本差异解释为稳定收益。

## 3. 逐题配对结果

### 对 E0 run1

- 两者都正确：17 题；
- E4-R0 恢复：2 题，`bird_1387`、`bird_1238`；
- E4-R0 退化：2 题，`bird_1340`、`bird_1460`；
- 两者都失败：29 题。

### 对 E0 run2

- 两者都正确：17 题；
- E4-R0 恢复：2 题，`bird_1472`、`bird_1350`；
- E4-R0 退化：3 题，`bird_1340`、`bird_1460`、`bird_1235`；
- 两者都失败：28 题。

E4-R0 没有在两次 E0 对照中都恢复同一批稳定失败题，`bird_1340` 和
`bird_1460` 还在两次比较中都发生退化。因此目前没有证据表明 capability gate
本身带来准确率增益。

## 4. 错误分布

E4-R0 的 31 条失败记录中，27 条为 `UNVERIFIED_FINAL`。这不是本实验要解决的
目标：E4-R0 关闭了 E1 的 strict verified-final，目的是隔离 capability gate，
因此该标签仍出现是预期结果。

其余可见错误各 1 条：

- `AGGREGATION_REASONING`：聚合或分组粒度不匹配；
- `OUTPUT_CONTRACT`：输出列数量不匹配；
- `SEMANTIC_REVIEW_REQUIRED`：过滤范围或表达式仍需语义复核；
- `TOOL_ERROR`：执行 observation 无法解析。

这表明 E4-R0 保持了 E0 的基础推理行为，尚未引入上下文外部化、程序化检索、
Planner 或 Leaf 分治，因此不能期待它解决当前主要语义错误。

## 5. 结论与下一步

E4-R0 通过了作为 RLM 消融父对照的校准：

- 题目集合、Prompt、few-shot 和模型配置与 E0 对齐；
- 结果在 E0 首 50 题的探索性范围内；
- 未观察到 capability 越权调用或递归调用；
- 没有明显准确率回归。

因此接受 `E4-R0` 作为后续实验的父基线。下一步只增加一个机制，运行
`E4-R1E`，把与 E0 相同的信息放入受控外部 context environment，先不加入
代码搜索、递归 Leaf 或 Planner。重点比较准确率、实际读取的 context、token、
LLM 调用和错误类别变化。
